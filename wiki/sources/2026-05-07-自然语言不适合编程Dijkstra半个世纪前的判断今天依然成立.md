# 自然语言不适合编程——Dijkstra 半个世纪前的判断，今天依然成立

> 原文链接：https://mp.weixin.qq.com/s/lmvR_H4QkgqNb3wvMTHHug

## 摘要

本文重读 Dijkstra 于 1978 年撰写的 EWD 667 备忘录《On the foolishness of "natural language programming"》，结合 2026 年 LLM 与 AI Coding Agent 的工程实践，分析「自然语言编程」这一理念被证伪还是被验证。

## 核心论点

Dijkstra 在**战术上输了**（机器确实能听懂人话），但在**战略上赢了**（本质难题只是迁徙而非消失）——所有认真做事的团队都在想方设法把自然语言「变回」形式化的东西。

## 关键洞察

### Dijkstra 的三层反驳（1978）

1. **界面选择不是零和游戏**：自然语言是「宽界面」，跨界面协作产生额外成本，界面越宽两侧越忙
2. **数学史的经验教训**：希腊数学受困于口述与图形化；阿拉伯代数曾尝试符号化又退回修辞式；现代科学的腾飞依赖于 Vieta、Descartes、Leibniz、Boole 等人用形式符号替换自然语言
3. **形式符号不是负担，而是特权**：母语的「自然」本质是让人容易说出表面通顺实则无意义的话；形式符号拒绝让人说出这类话

### 今天的实践验证

| 实践 | 实际在做的事 |
|------|-------------|
| XML 标签包裹输入（`<context>...</context>`） | 在自然语言中重建语法边界 |
| 要求模型输出 JSON 而非自由文本 | 在输出端重建形式结构 |
| Spec-driven development | 在指令端重建形式化契约 |
| Few-shot examples | 通过样本压缩指令的歧义空间 |
| Tool use / Function calling | 将关键动作从「描述」退回为「调用」 |
| Constitutional AI | 在价值层补上形式化约束 |

### 三条预言级精度

1. **跨界面协作产生额外工作量** → 对应今天的 Prompt Engineering 产业链
2. **母语的自然易导致无意义输出** → 对应 LLM 幻觉问题
3. **新文盲症** → 对应以为会打字就会写 Prompt 的认知偏差

### 三条启示

1. 不要将 LLM 的「能听懂人话」视作免费午餐
2. 写好 Prompt 的核心能力是**形式化思维**，不是文采
3. 真正稀缺的能力是在**自然语言与形式符号之间自由切换**
