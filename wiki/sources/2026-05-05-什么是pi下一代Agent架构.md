# 什么是 Pi？下一代 Agent 架构

> 极简 AI 编程代理：只有四个核心能力，却成了 OpenClaw 的底层引擎

关于 [[Pi]] 的架构哲学和设计理念，请参阅实体页面。

## 核心论点

Pi 是游戏引擎开发者 Mario Zechner 打造的极简 AI 编程代理。它的核心哲学是"不做什么"——刻意省略 MCP 协议、子代理、计划模式、权限弹窗、内置任务列表等功能，把"加功能"这件事交还给用户。

## 关键论述

### Pi 的四个核心能力

Pi 极其克制，只有四个工具：
- **Read** — 读取项目代码文件
- **Edit/Write** — 直接修改或创建文件
- **Bash** — 执行命令、跑测试、安装依赖
- **多模型支持** — 支持 15+ AI 模型（Anthropic、OpenAI、Google、本地 Ollama 等）

### "不做什么"才是设计哲学

Pi 官网有专门的 "What we didn't build" 板块：
- 没有 MCP 协议支持 — 写技术文档告诉 AI 怎么做就够了
- 没有子代理 — 用 tmux 多开终端自己管理
- 没有权限确认弹窗 — 跑在容器里就好
- 没有计划模式 — 新建 TODO.md 文件
- 没有内置任务列表 — 任务列表会占用模型上下文

### 树状对话历史

与绝大多数 AI 工具的线性对话不同，Pi 把对话历史做成树状结构。可以在任意历史节点上"分叉"，往不同方向试验，再回来对比结果。思路接近 Git 的分支管理。

|### Pi 与 OpenClaw 的关系

OpenClaw 是 Pi 的"上层建筑"（关于 [[OpenClaw]] 的完整架构请参阅实体页面）：
- OpenClaw 使用 Pi 的 RPC 模式运行
- Pi 负责"怎么执行"
- OpenClaw 负责"从哪里来、到哪里去"（多渠道接入、多 Agent 调度）

### Pi 与其他工具对比

| 功能 | Pi | Claude Code / Cursor |
|------|-----|---------------------|
| 核心哲学 | 极简，可扩展 | 功能完整 |
| 支持 AI 模型 | 15+ 个 | 通常绑定 1 家 |
| 树状对话历史 | 原生支持 | 不支持 |
| TypeScript 扩展 | 50+ 示例 | 通常不支持 |

## 主要观点

1. **Pi 更像一把好的厨刀，而不是多功能料理机** — 需要懂得怎么用，但更灵活
2. **工具应该适应你的工作流，而不是反过来** — 这是 Pi 的核心立场
3. **用 AI 扩展 AI 工具** — 如果需要某个功能，自己用 TypeScript 写一个扩展，或者让 Pi 帮你写

## 结论

Pi 选择了 Unix 哲学的方向——把能砍掉的全砍掉，把剩下的做扎实，然后把"加功能"交还给用户。在 AI 工具这个赛道坚持这个方向，并被 OpenClaw 采用，说明这种克制是有效的。
