# Anthropic 最新博客：MCP 没死，它又来了

> Anthropic 发布官方博客回应社区对 MCP 的批评，提出 Tool Search（减少 85% token）和程序化工具调用两大解法，并指出 MCP 的真正定位：云端生产环境的标准化接入层，与 CLI + Skills 形成互补格局。

## 背景：社区对 MCP 的三大批评

1. **Token 昂贵**：ScaleKit benchmark 显示 CLI 比 MCP 便宜 10-32 倍
2. **Schema 臃肿**：GitHub MCP 服务器带 43 个工具定义，每次对话都要全部塞进上下文
3. **不可组合**：Perplexity CTO 指出 72% 上下文窗口被 MCP 占掉

## Anthropic 的回应

### 三条路并存

Agent 连接外部系统有三条路：
- **直连 API**：适合简单、一对一场景
- **CLI**：本地/沙箱环境更合适，Agent 天生说命令行语言
- **MCP**：云端生产环境，跨客户端标准化

### 关键数据

- MCP SDK 月下载量从年初 1 亿涨到 3 亿
- Claude、ChatGPT、Cursor、VS Code 一个 MCP 服务器全覆盖

## Token 问题的两大解法

### 1. Tool Search（工具搜索）

**传统方式**：把所有工具定义一股脑塞进上下文
- 43 个工具，55,000 tokens

**Tool Search 方式**：按需加载
- Agent 先描述想做什么，系统搜索匹配的工具
- 只把匹配的几个工具拉进来
- **效果：工具定义 token 减少 85% 以上**

**分组建议**：按意图分组，而非按 API 分组

### 2. 程序化工具调用

- 工具返回结果不再直接丢回模型
- 在代码执行沙箱里处理
- Agent 可循环、过滤、聚合
- **效果：复杂多步工作流减少约 37% token 消耗**

### 效果对比

| 方案 | GitHub 仓库语言查询 token | 与 CLI 差距 |
|------|---------------------------|-------------|
| 传统 MCP | 44,026 | 32 倍 |
| Tool Search 后 | ~10,000 | 7 倍 |

## Cloudflare 案例：代码编排模式

- MCP 服务器覆盖 2,500 个 API 端点
- 只暴露 2 个工具：`search` 和 `execute`
- Agent 先用 search 找到需要的 API，然后写脚本通过 execute 在服务端沙箱执行
- 整个工具定义只占约 1K tokens

**这个模式本质上是把 CLI 的哲学搬进了 MCP 协议里**

## MCP + Skills 互补

- **MCP 管「能力」**：让 Agent 连接 Snowflake、Databricks、BigQuery
- **Skills 管「编排」**：告诉 Agent 怎么用这些连接完成任务
- Claude 数据插件：10 个 Skills + 8 个 MCP servers 打包

## 结论：MCP 的地盘

| 场景 | 方案 |
|------|------|
| 本地开发环境 | CLI + Skills，轻量、快速、上下文干净 |
| 云端生产环境 | MCP + Skills，标准化、跨平台、认证完备 |
| 简单场景 | 直连 API |

**MCP 没有死，它正在成为云端 Agent 的标准化接入层。**
