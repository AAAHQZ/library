# 十年老技术开发的 AI Agent 探索之路

> 从 4 个终端的手忙脚乱，到 24 小时无人值守的 Agent 系统——一条完整的 AI Agent 工程化实践路径。

## 核心观点

1. **人是瓶颈**：人脑并发上限 4-6 个 Agent 窗口，解决方案不是让 AI 替代人，而是让系统不再依赖人的实时在场
2. **80% 的 AI 需求不需要 AI**：能用 10 行 Bash 解决的别折腾 AI。决策层级：目标 → 代码 → CLI → Prompt → Agent
3. **Vibe Coding 翻车**：先易后难，前三天省掉的设计时间以 10 倍 debug 时间还回来
4. **SDD（Spec-Driven Development）**：spec.md → plan.md → tasks.md，让 AI 每一步都有据可查
5. **自建调度层**：文件系统 + 轮询调度 + JSON 状态管理，从 4 终端到 20-30 并发任务
6. **Agent 自举**：在 design docs + SDD + constitution.md 基础上，Agent 可以自己修复自己的 bug
7. **脚手架 > 模型**：模型升级 +300% 成本/+20% 效果；脚手架升级 +50% 成本/+200% 效果
8. **Agent 系统 5 层地基**：目标表达、能力单元、运行时状态、治理边界、评估反馈
9. **协议层收敛**：Responses API / MCP / A2A/ Agentic Workflows — 从框架之争转向协议+Runtime+Control Plane 之争
10. **Task-Driven → Goal-Driven**：先让执行可复盘 → 可重复 → 可规模化 → 可有限自主

## 引用原则

> 代码优先于 Prompt。能用 10 行 Bash 解决的，别折腾 AI。

> Vibe Coding 是先易后难。SDD 是先难后易。大道如夷，而民好径。

> 捷径的尽头是弯路，大道的尽头是自由。

> 留痕不是为了 debug，而是为了进化。

> 脚手架 > 模型。优先投资脚手架，而不是追最新最贵的模型。

> 垃圾的思考乘以强大的模型，等于精美的垃圾。

> 真正的跃迁，不是让 AI 多做几个步骤，而是让人退出微观调度。

> 增强自我，而非取代自我。
