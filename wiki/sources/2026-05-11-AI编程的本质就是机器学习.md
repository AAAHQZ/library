## 核心论点：AI 编程就是机器学习

### 实践方式
作者使用 Codex /goal 或 Claude Code 的姿势：
1. 写好需求描述（目标）
2. 定义好验收标准及测试用例（约束/验证集）
3. 提供好环境
4. 让 agent 自己跑：改代码 → 跑测试 → 看结果评估 → 再改，循环往复直到全部完成

> 案例：一次跑了 10+ 小时，代码全跑完后只需抽查表结构、页面样式和交互，符合预期就合并。代码逐行审查已越来越少见。

### 关键等式（原文核心）

| 传统编程 | 机器学习 |
|----------|----------|
| 需求文档（PRD） | loss function（损失函数） |
| 测试用例 | 验证集 |
| coding agent | 模型架构 |
| Agent 每一轮迭代 | training step（训练步） |
| 产出的代码库 | 训练好的模型权重 |

> **一旦接受了这个等号**，你花时间的地方不再是写代码本身，而是：
> - 我的优化目标定对了吗？
> - 我的验证标准够不够严？
> - 我的约束条件会不会让 agent 找到某个投机取巧的捷径（测试全过但逻辑是歪的）？

### ML 领域的同等坑

- **过拟合**：测试全过但实际逻辑偏离
- **数据泄露**：验证标准被污染
- **概念漂移**：需求本身在迭代过程中悄然改变

### Chollet 的终极问题

> AI 编程的 Keras 会是什么？什么样的高层抽象，能让人用最少的认知成本，去驾驭这个"训练"过程？
