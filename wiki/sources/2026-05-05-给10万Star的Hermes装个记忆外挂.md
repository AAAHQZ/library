# 给 10 万 Star 的 Hermes 装个记忆外挂

> Hermes 让 Agent 能干活，MemOS 让它越干越聪明。

## Hermes Agent 是什么

|[[Hermes-Agent]] 是 Nous Research 开源的一个自主 AI Agent 框架（GitHub 10 万+ Star），核心思路：**一个部署在你自己设备上的 AI Agent，用得越久越强**。

## 原生问题：记得住，但记得乱

**问题场景**：用户说"在减肥，每天 1800 大卡"，一周后说"放弃减肥了"，下次规划周末时 Hermes 还是推荐低卡食谱——因为两条记忆都在，分不清哪个是最新的。

**根本原因**：
- 原生把每轮对话直接存 SQLite，检索时做文本匹配
- 同一信息在不同对话里反复出现，导致重复条目大量堆积
- 信噪比越来越差

## MemOS 插件解决方案

关于 [[MemOS]] 的概念和架构，请参阅实体页面。

MemTensor 团队推出的 Hermes 本地记忆插件，完全跑在本地，数据不用上传云服务。

### 存储端：记忆存得聪明

**四步处理**：语义分片 → LLM 摘要 → 向量化 → 智能去重

**智能去重**是关键：不是简单文本比对，而是把当前要存的内容和已有相似记忆做对比，让 LLM 判断是重复的、需要更新的、还是全新的。

**效果**：更新和原记录合并成一条，保留合并历史，记忆库始终保持干净。

### 检索端：记忆找得准

**混合检索引擎**：全文搜索 + 向量语义搜索双通道同时跑

加上**融合排序、多样性去重、时间衰减、相关性过滤**四层处理。

**效果**：语义通道能把"上次推荐了什么好吃的地方"即使原文没有"推荐"和"好吃"关键词也能捞出来。

**预检索机制**：每轮对话开始时自动用最新消息做预检索，把相关记忆注入上下文。

## 技能进化

- **三级独立模型配置**：Embedding 用轻量模型、摘要用中等模型、技能生成用最强模型
- **规则过滤 + LLM 评估**：只有可重复、有价值的任务才会生成技能
- **降级机制**：技能模型挂了 → 自动降到摘要模型 → 再挂了降到 Hermes 原生模型

## 多 Agent 协同

1. **同一机器多 Agent**：独立记忆空间 + 共享公共记忆和技能
2. **跨机器 Hub-Client 架构**：私有数据始终留在本地，只有明确共享的内容对团队可见

## 管理面板

装完插件后访问 `http://127.0.0.1:18901`，7 个管理页面：记忆浏览/搜索、任务管理、Skill 管理、分析统计、工具调用日志、数据导入、在线配置。密码保护。

## 安装

```bash
curl -fsSL https://raw.githubusercontent.com/MemTensor/MemOS/openclaw-local-plugin-20260408/apps/memos-local-plugin/install.sh | bash
```

前置条件：Node.js ≥ 18、Python 3、Hermes Agent 已装好。
