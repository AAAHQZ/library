# AI Agent 框架选型与工程实践

> 本文系统阐述 AI Agent 的核心框架理论（ReAct/Plan-and-Execute/Reflection），对比主流框架（LangChain/LlamaIndex/AutoGen/CrewAI/LangGraph/Semantic Kernel），并通过极简代码示例展示 Agent 框架的核心工程实现：LLM Call + Tools Call + Context Engineering，以及贯穿其中的 Agent Loop 机制。

## 核心概念

### Agent 基础理论

**Agent = Reasoning + Acting**

AI 智能体是使用 AI 来实现目标并代表用户完成任务的软件系统，表现出推理、规划和记忆能力，具有自主性。

### 三种核心模式

1. **ReAct 模式**（2022年 Yao 等人提出）
   - 核心：将推理（Reasoning）和行动（Acting）相结合
   - 循环：推理 → 执行 → 观察
   - 优点：弥补了 CoT 缺少外部世界交互的缺陷

2. **Plan-and-Execute 模式**（2023年 LangChain 提出）
   - 核心：先制定完整的分步计划，再按步骤执行
   - 适合：复杂且任务关系明确的长期任务
   - 缺点：缺乏动态调整能力

3. **Reflection 模式**
   - 代表论文：Reflexion、Self-Refine、CRITIC
   - 核心：通过语言反馈或外部工具验证来自我修正

### 主流框架对比

| 框架 | 特点 | 适用场景 |
|------|------|----------|
| LangChain | 最成熟，工具链丰富 | 快速构建复杂 AI 应用 |
| LlamaIndex | 专注数据索引和检索 | RAG 场景 |
| AutoGen | 微软多 Agent 协作 | 复杂任务分解 |
| CrewAI | 角色扮演型协作 | 模拟团队协作 |
| LangGraph | 状态图框架 | 复杂流程控制 |
| Semantic Kernel | 微软轻量级 | .NET 生态 |

### Agent 框架核心

**上下文工程（Context Engineering）是 Agent 框架智能的核心**

Manus 的经验教训：
- 使用文件系统作为上下文（如 OpenClaw 的 SOUL.md/TOOLS.md/MEMORY.md）
- 编程是解决通用问题的普适方法（CodeAct 理念）

**Agent 框架三大部分：**
1. **LLM Call**：API 管理，兼容各大 LLM 厂商
2. **Tools Call**：从 Function Call 到 MCP 到 Skill
3. **Context Engineering**：狭义指 Prompt 工程，广义包含工具调用

### Agent Loop

Agent Loop 是 Agent 框架的核心运行机制，本质是一个 While 循环：
```
初始上下文 → Agent Loop → 读取上下文 → 思考 → 决定行动 → 执行工具 → 获得结果 → 结果追加到上下文 → 循环或结束
```

- **Agent 框架核心 = 在 Agent Loop 中设计如何管理上下文**

关于 [[AI-Agent]] 的核心概念和范式转变，请参阅实体页面。

## 工程实践

### 极简 Agent 框架架构

```
┌─────────────────────────────────────────────┐
│  User Interface（CLI REPL）                  │
├─────────────────────────────────────────────┤
│  Agent Loop Core                            │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ LLM Call │─▶│Tool Call │─▶│Tool Exec │   │
│  └──────────┘  └──────────┘  └──────────┘   │
│        │                                      │
│        ◀─────────────────────────────────────┘
│        (Tool Results Feedback)               │
├─────────────────────────────────────────────┤
│  Tools Registry                             │
│  shell_exec | file_read | file_write | python_exec │
└─────────────────────────────────────────────┘
```

### 四个核心工具

- `shell_exec`：执行 shell 命令
- `file_read`：读取文件内容
- `file_write`：写入文件（自动创建目录）
- `python_exec`：在子进程中执行 Python 代码

### 关键洞察

1. **代码即工具**：[[OpenClaw]] 的底层 Agent Core（[[Pi]] Agent）只有 4 个工具：Read、Write、Edit、Shell
2. **上下文工程是低垂的果实**：在不提供任何 Context 的情况下，最先进的模型仅能解决不到 1% 的任务
3. **Agent 应用内，上下文工程是智能的核心**（短期/长期记忆、用户 Session 管理、动态 RAG 等）
