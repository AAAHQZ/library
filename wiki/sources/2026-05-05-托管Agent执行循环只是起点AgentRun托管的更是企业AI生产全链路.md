# 托管 Agent 执行循环只是起点，AgentRun 托管的更是企业 AI 生产全链路

> 阿里云 AgentRun 作为一站式 Agentic AI 基础设施平台，提供从开发、部署到运维的全生命周期管理，平均 TCO 降低 60%。

## 核心概念

### 托管式 Agent 行业背景

AI Agent 正从技术概念快步走向生产应用。从"跑通 Demo"到"稳定上线"每一步都是对基础设施的真实考验——更聪明的模型解决不了这道鸿沟。

**行业共识**：Anthropic Claude Managed Agents 印证了托管式 Agent 方向，核心围绕四个概念：
- **Agent 定义**：声明模型、系统提示词、可用工具，一次创建多会话复用
- **运行环境**：容器模板，定义网络策略、文件系统等隔离边界
- **会话**：Agent 与运行环境绑定，代表一次完整任务执行
- **事件流**：会话内消息通道，实时推送用户输入、模型回复、工具调用

### AgentRun 核心抽象

- **超级 Agent**：内置 Agent Loop 的可执行对象
- **Agent Runtime 与 Sandbox**：运行环境和隔离沙箱，底层基于阿里云函数计算 FC
- **会话管理**：持有上下文、中间状态、事件流
- **MCP 工具 与 Skills 市场**：工具级扩展 + 任务级扩展两条路径
- **模型服务 ModelService**：模型代理层，解耦 Agent 逻辑和模型选择
- **记忆与知识库**：长期记忆和 RAG 检索作为 Agent 常驻上下文

## 差异化能力

### 运行时与沙箱

基于阿里云函数计算 FC，毫秒级启动，内核级隔离。能装下代码解释器（Jupyter Kernel）、浏览器自动化（CDP over WebSocket）、Computer Use（桌面级 GUI）。

**优势**：
- 多租户隔离更彻底（内核级，不存在容器逃逸风险面）
- 沙箱能装下更重的能力（资源控制和兼容性空间更大）

### 厂商无关的模型服务

AgentRun 不绑定特定模型厂商，可自由切换：通义千问、DeepSeek、OpenAI、自部署开源模型。

**使用场景**：
- 成本敏感场景优先走小模型，必要时 Fallback 大模型
- 代码生成、长文本理解、多模态处理各路由到适合的模型
- 合规场景对数据出境有硬性要求，只能使用境内或自部署模型

### MCP 与 Skills

两条互补路径：
- **MCP**：原生支持标准协议，生态里已有 MCP Server 直接接入，粒度小、组合自由
- **Skills 市场**：任务级能力单元，上千种预置技能（网页检索、文档解析、数据可视化、SaaS API 对接）

**私有 Skills 托管**：支持上传 Python/Node.js 包，或用 Markdown 声明式语法描述；AI 自动生成脚手架；代码安全扫描；可挂到独立 Sandbox 运行（异常被 MicroVM 隔离）。

### 会话与记忆

- **短期记忆**：跟随会话
- **长期记忆**：跨会话保留用户偏好、历史结论、跨任务知识
- **知识库**：承载 RAG 场景完整链路（文档切片、向量化、检索召回）

## 企业级能力

### 数据不出域：VPC 与专有网络

三种模式：
- **PUBLIC**：完全公网访问
- **PRIVATE**：完全 VPC 内，不接公网
- **混合**：既能访问公网，也能打通 VPC

Agent 可直接访问企业 VPC 内数据库、中间件、内部 API，请求链路在阿里云内部闭环。

### 统一凭证管理

AgentRun 把 Credential 作为一级资源，支持 OAuth2、API Key、JWT、Basic Auth、AK-SK、自定义 Header 等。统一创建、绑定到工具或技能、按需禁用和轮换，传输与存储全程加密。

### RAM 集成与多租户

对接阿里云 RAM 权限体系，不同子账号/团队看到的 Agent、工具、模型资源各自隔离，权限粒度可细到资源级。

### 可观测性：OpenTelemetry 全链路

- 每次模型调用：耗时、Token 消耗、命中模型
- 每个工具调用：入参、出参、延迟
- 任务整体成本：按 Agent、会话、工具维度归因

### 模型治理

- 多 Key 负载均衡，避免单个 Key 触发限流
- 主备模型 Fallback，一个不可用时自动切换
- Token 级限流和并发控制，防止成本失控
- 调用级成本归因，追溯每次推理费用

### Serverless 原生弹性

构建在函数计算之上，没请求时缩到 0 不计费，有流量时按秒计费，冷启动百毫秒级。
