# AI 经济学

> 本文由 Ed Zitron 文章《AI's Economics Don't Make Sense》解析生成，原始文档：`/data/data/com.termux/files/home/library/raw/wechat/2026-04-29-AI的经济账根本算不通.md`

## 概述

AI 经济学指围绕大语言模型（LLM）服务的成本结构、定价模式和商业可持续性问题。该领域核心矛盾在于：**LLM 的按量计费成本与面向用户的月费订阅模式之间存在根本性错配**，导致 AI 公司持续亏损，且这种模式不可持续。

## 核心概念

### Token 经济学

| 概念 | 说明 |
|------|------|
| **Token** | AI 模型处理的最小单位，约等于 3/4 个英文单词 |
| **输入 Token** | 用户发送给模型的 Prompt |
| **输出 Token** | 模型生成的回复 |
| **上下文窗口 (Context Window)** | 模型一次能处理的最大 Token 数量 |
| **百万 Token 价格** | 输入约 $5/百万 Token；输出约 $25/百万 Token |
| **代码换算** | 100 万 Token ≈ 50,000 行代码 |

**关键问题**：用户对 Token 消耗毫无感知，月费订阅掩盖了真实使用成本。

### 推理成本悖论

**预期**：算力成本随时间下降
**现实**：
- 某些简单模型价格下降
- 推理模型（Reasoning Models）消耗更多 Token
- 推理成本反而随时间上升

## 核心矛盾

### 订阅模式 vs 实际成本

```
Uber 类比（错误）：
用户付 $20/月 → 坐 100 次车，Uber 承担油费 → 听起来合理

AI 实际：
用户付 $20/月 → 可烧掉 $160+ 的 Token → 相当于 Uber 报销每加仑 $150 汽油
```

### 月费订阅的不适用性

| 行业 | 成本特征 | 适合订阅？ |
|------|----------|------------|
| 健身房 | 器材磨损、水电、人工成本稳定 | ✅ 适合 |
| Google Workspace | 存储/访问成本低，相对稳定 | ✅ 适合 |
| LLM 服务 | 成本剧烈波动，取决于模型、任务、上下文 | ❌ 不适合 |

**问题**：用户使用模式差异极大，但订阅价格相同：
- 轻度用户：偶尔搜索 → 成本极低
- 重度用户：重构代码库、制作 PPT → 成本可能超出月费 10 倍以上

## 商业模式危机

### 次贷式 AI 危机 (Subprime AI Crisis)

Ed Zitron 提出的概念：整个科技行业以极低折扣出售高度集中、由大公司补贴的技术。总有一天：
1. 烧钱速度追上公司
2. 涨价或推出苛刻新费率
3. 连最忠诚的企业客户也无法证明支出合理

### 谎言金字塔

1. **AI 公司** → 订阅模式掩盖真实成本
2. **媒体** → 报道时不体验真实 Token 成本
3. **企业** → 员工被要求"尽可能多用 AI"
4. **结果** → 没有人真正了解实际支出

### CFO 警告信号

OpenAI CFO Sarah Friar 明确表示：
- 担心收入增长不够快，无法支付计算合同
- 公司"还没准备好达到上市公司严格报告标准"
- 增长正在放缓，却需要在 2030 年前实现 10 倍增长
