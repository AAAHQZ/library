# 知识对账（Knowledge Reconciliation）

> EventHouse 提出的机制，通过 Knowledge Wiki 让人和 Agent 之间建立可审查、可迭代的知识对齐机制，确保 Agent 对取数逻辑和语义的理解正确，避免黑盒操作。

## 简介

知识对账（Knowledge Reconciliation）是基于 DIKW（Data-Information-Knowledge-Wisdom）模型设计的机制。在 EventHouse 实践中，一方面基于统一 Catalog 中的数据定义、Schema 描述和语义信息，另一方面结合客户对 Agent 的业务设定（SOUL、Prompt、Gold Sample、Benchmark 等），生成一份可读、可审查、可持续迭代的 Knowledge Wiki。这份 Wiki 既能被 Agent 消费，也能被人类专家审阅。

## 详细内容

### 核心价值

让 Agent 不只是"连上数据"，而是真正开始"理解数据"。通过建立可审查的知识对账机制，解决 Agent 落地中最难的问题之一：如何确保 Agent 对业务数据的理解与人类专家一致。

### DIKW 模型在知识对账中的应用

- **数据（Data）**：客观事实的原始记录
- **信息（Information）**：被赋予语境与结构的数据，回答"What"（发生了什么）
- **知识（Knowledge）**：信息基础上提炼的规则、经验和方法，回答"How"（如何找、理解和用信息）
- **智慧（Wisdom）**：在明确目标与价值取向的前提下，在复杂情境中对知识进行综合运用和权衡
