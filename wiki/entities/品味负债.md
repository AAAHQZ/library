# 品味负债（Taste Debt）

> Harness 搭建初期在约束设计、评估标准、权限策略、知识治理上做的隐性妥协，不报编译错误，不在 benchmark 上暴露，只在真实业务摩擦中慢慢显影。

## 简介

品味负债是与技术债平行的概念。技术债是工程师为了快速上线在代码质量、架构设计上做的妥协；品味负债是团队在 Harness 搭建初期，为了"先跑起来"，在约束设计、评估标准、权限策略、知识治理上做的隐性妥协。它不阻止你出发，但会悄悄改变你的目的地。

Gartner 2026 Q1 AI Agent 部署报告显示：67% 的企业 Harness 系统存在中度以上品味负债。平均修复成本是初期设计的 4.3 倍，且随业务复杂度呈指数级放大。

## 四大病灶

### 1. 约束僵化（Constraint Rigidity）
初期为防 Agent"脱缰"设下过密规则墙，Agent 变成提线木偶。本质是用静态规则应对动态业务。
- **案例**：某电商 Agent 设 127 条硬规则，Q2 促销调整后 Agent 因规则冲突频繁 fallback，人工审核工单暴涨 300%

### 2. 评估失真（Evaluation Mismatch）
评估器停留在"任务完成率""代码通过率"等显性指标，真实场景中的"摩擦系数""维护成本"被系统性忽略。本质是优化了错误的目标。
- **案例**：某金融 Agent Terminal Bench 得分 Top 5，但生产环境首次可合并率仅 28%

### 3. 权限摩擦（Permission Friction）
权限体系"一刀切"审批流。低风险操作走重流程，高风险操作靠运气拦截。本质是缺乏动态置信度管理。
- **案例**：某运维 Agent 每次调用生产数据库均需双人审批，3 个月后 Agent 记忆断裂率升至 41%

### 4. 知识腐败（Knowledge Decay）
Knowledge 模块缺乏版本管理、时效标记、冲突消解机制，Agent 基于过时信息决策。本质是静态堆砌必然导致上下文污染。
- **案例**：某 SaaS 团队 800 页产品文档，6 个月后 32% 引用指向已下线功能

## 复利利息

品味负债不一次性爆发，而是以复利方式啃噬工程效能：
1. **干预率攀升**：人工干预率每上升 10%，开发者认知负荷增加 23%，上下文切换成本上升 31%
2. **上下文膨胀**：为弥补约束不足往 Context 塞更多内容，有效信号占比从 68% 跌至 29%
3. **迭代阻力**：修复成本随部署时长呈 1.8^n 指数增长
4. **团队倦怠**：工程师从"驾驭者"退化为"救火员"，创新力归零

## 清算指南

1. **定期品味审计**（Taste Audit）：每季度聚焦约束弹性、评估对齐、权限呼吸感、知识鲜活度
2. **动态约束调优**：场景分级 + 置信度路由 + 反事实测试
3. **摩擦感知评估**：引入人工摩擦系数、上下文健康度、长期稳定性指标
4. **人在回路的品味校准**：周级品味复盘 + 约束众包 + 品味资产库

## 味觉——品味负债的正面（工程护城河）

品味负债的反面是**工程味觉**——在效率与韧性、规则与弹性、正确与对味之间精准校准的隐性判断力（[[sources/2026-05-10-味觉：Harness时代，最后的工程护城河]]）。

味觉的三层具象与品味负债的四大病灶形成镜像对照：

| 味觉维度 | 对应病灶 | 核心原则 |
|----------|----------|----------|
| 约束的颗粒度：懂得克制 | 约束僵化 | "只定义绝不能做什么"比规定怎么做更有效 |
| 评估的隐性权重：摩擦系数 | 评估失真 | 单纯追求完成率的 Harness 崩溃率高出 3.2 倍 |
| 权限的呼吸感：信任曲线 | 权限摩擦 | 沙箱不是牢笼是训练场 |

**数据**：同一模型两种 Harness，产品留存率差 3.8 倍；多维度味觉评估器使驳回率↓64%、故障率↓89%。
