# 24h 打工人

> 一个真正能 24 小时无人值守运行的 Agent 系统，由资深开发者 zhiyuanfu（@joefu）构建。

## 背景

开发者从手动管理 4-6 个 AI 终端（Codex、Gemini-CLI、Claude、Cursor）的混乱体验出发，决心构建一个不依赖人实时在场的自动化系统。

## 核心架构

**四个字：文件 + 轮询**

- **文件系统存储 + 轮询调度 + JSON 状态管理** — 非教条选择，而是最利于 AI 排查问题的实际考虑
- **SDD 流程**：spec.md → plan.md → tasks.md 三级文档驱动
- **智能并发**：组间并发（前后端同时跑）、组内串行（避免文件冲突）、失败隔离
- **工具失败自动切换**：Tool Prober 定时探测可用性，codex 配额耗尽自动切 gemini-cli，再切 claude
- **Constitution.md**：架构约束文件，定义代码组织、模块边界、命名规则

## 关键成就

- 从 4 个终端的手忙脚乱到 20-30 个并发任务的稳定执行
- Agent 通过自身 SDD 流程实现了"自己修复自己的 bug"（自举）
- 使用非最贵模型，靠系统架构获得远超直接用顶级模型的效果

## 决策原则

选择 CLI 而非 API 的原因：Codex、Gemini-CLI、Claude-Code 已内置了读文件、改代码、跑命令的能力。需要造的不是另一个 Agent，而是管理 Agent 的调度层。
