# Agent 隔离安全

## 概念概述

**Agent 隔离安全**是针对 AI Agent 安全防护的核心理念与实践，重点在于**策略（Policy）而非隔离（Isolation）**。传统观念认为需要用容器、微虚拟机等重量级运行时构建隔离层，但实际场景中 Agent 不是攻击者，真正需要应对的是 Prompt 污染和误操作。

## 核心观点

1. **Agent 不是攻击者**：Agent 是按 prompt 执行的模型，没有自主意图，不会主动尝试突破沙箱
2. **隔离 ≠ 安全**：即使 Agent 逃不出容器，仍可能读取敏感文件
3. **大多数场景不需要 root**：按需授权即可满足需求
4. **工具级隔离优于全局沙箱**：每个工具调用单独设立沙箱，遵循最小权限原则

## 关键技术

### Landlock

Linux 6.12+ (ABI v6) 支持的内核安全接口，提供：
- 文件系统访问控制（路径级白名单）
- TCP 端口限制
- IPC 和信号隔离

与容器/微虚拟机不同，Landlock 在进程内部施加约束，无需特权守护进程。

### seccomp-bpf

- 最低内核版本：Linux 3.5+
- 功能：系统调用过滤
- 设置 `PR_SET_NO_NEW_PRIVS` 后由进程自行施加

### 用户命名空间

- 最低内核版本：Linux 3.8+
- 功能：可选的 UID 映射，非特权用户即可创建

## 相关概念

| 概念 | 说明 |
|------|------|
| Prompt注入 | 通过污染上下文让 Agent 执行恶意指令 |
| 沙箱安全 | 传统基于容器的安全模型 |
| Sandlock | 策略优先的轻量 Agent 沙箱实现 |
| 最小权限原则 | 每次工具调用单独设沙箱 |

## 与传统方案的对比

| 特性 | Sandlock | 容器 | 微虚拟机 |
|------|----------|------|----------|
| 需要 root | 否 | 是 | 是 |
| 启动时间 | ~5ms | ~200ms | ~100ms |
| 文件系统隔离 | Landlock + seccomp | Overlay | 块级 |
| 工具级隔离 | 是 | 否 | 否 |

## 核心策略原则

1. **默认全部拒绝**：路径、网络、能力默认不可访问
2. **按路径开放白名单**：精确到 `/src` 只读、`/src/output` 写权限
3. **按工具做限制**：浏览工具可联网但不能写磁盘
4. **内核层强制执行**：依赖 Landlock 和 seccomp
