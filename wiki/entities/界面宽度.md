# 界面宽度

> Dijkstra 提出的界面理论概念：两个系统之间传递信息的约定边界的「宽度」。

## 核心原理

- **窄界面**：信息量小、歧义少、两侧协作成本低
- **宽界面**：信息量大、歧义多、两侧协作成本高
- **工程实践中偏好窄接口**的根本原因：跨界面协作会产生额外工作量，界面越宽，两侧越忙

## 在 AI 时代的应用

| 场景 | 宽界面 | 窄界面（收窄手段） |
|------|--------|-------------------|
| Prompt 输入 | 自由自然语言 | XML 标签、格式化结构 |
| 模型输出 | 自由文本 | JSON Schema、Structured Output |
| 任务指令 | 模糊需求描述 | Spec 文档、形式化契约 |
| 工具调用 | 描述意图 | Function Calling、Tool Schema |
| 行为约束 | 隐含期望 | Constitutional AI、显式规则清单 |
