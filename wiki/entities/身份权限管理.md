# 身份权限管理

## 概述

身份权限管理（Identity & Access Management, IAM）是AI Agent时代安全的核心防线。本文围绕OpenClaw安全事件，深入分析传统IAM在Agent场景下的失效原因，并探讨现代Agent IAM应具备的核心要素。

## 核心概念

### OpenClaw
- **类型**：自主智能体（Autonomous Agent）框架
- **风险**：用户往往授予其"上帝模式"（God Mode）——文件读取、代码执行、互联网访问等广泛权限
- **安全事件**：
  - CVE-2026-25253：远程代码执行漏洞
  - ClawHavoc投毒事件：2,857个Skills中发现341个恶意Skills
  - 近千个公网暴露实例无身份认证

### IAM（Identity & Access Management）
- **定义**：身份和访问管理，成为Agent时代的唯一防线
- **传统IAM局限**：
  - 为人类用户和静态服务设计
  - 身份会发生传递（多层代理/外包链）
  - 静态权限与动态上下文冲突
  - 权限颗粒度不足
  - 密钥容易泄漏

### 零信任（Zero Trust）
- **核心理念**：在自主Agent时代，传统网络边界已不复存在，身份成为唯一的安全边界
- **关键原则**：永不信任，始终验证

### 致命三要素（Lethal Trifecta）
安全研究员Simon Willison提出的Agent漏洞标准框架：
1. **访问私有数据**（Access to Private Data）：读取邮件、文档、数据库、代码库
2. **外部通信能力**（External Communication）：调用外部API的合法通道
3. **处理不可信内容**（Exposure to Untrusted Content）：接收并处理外部数据

## Agent IAM核心要素

### 1. 身份传播（Identity Propagation）
- **定义**：确保人类用户身份上下文能穿透代理层，传递到Agent所调用的后端服务
- **目标**：防止"混淆代理人"（Confused Deputy）攻击
- **作用**：解决"我是谁"的问题，防止Agent身份被滥用为通用钥匙

### 2. 无秘钥验证（Secretless Auth）
- **定义**：将"密钥的持有"与"密钥的使用"彻底解耦
- **实现**：密钥存储在Agent无法访问的外部安全环境，Agent只持有引用标识符
- **目标**：使用短生命周期的动态密钥
- **作用**：解决"凭证在哪里"的问题，消除Agent暴露大量密钥的风险

### 3. 上下文感知（Context Awareness）
- **定义**：权限决策基于Agent运行时完整性与会话状态
- **验证内容**：
  - 执行环境是否受信任（如AWS Nitro Enclave, Confidential VM）
  - SessionAttributes是否包含完成操作所需的前序状态
- **作用**：解决"环境与逻辑状态是否可信"的问题

### 4. 意图感知授权（Intent-Aware Authorization）
- **定义**：深入语义层面的授权，检查Agent"能不能"做某事，更检查"为什么"要做
- **作用**：防御提示词注入与逻辑越狱
- **区别**：传统IAM无法理解语义，只有意图感知能防御逻辑层面的攻击

## 市场主流方案

### AWS AgentCore Identity
- **身份传播**：Token置换，含委托关系及用户身份信息
- **无秘钥验证**：Outbound Gateway + Token Vault隔离托管
- **上下文感知**：基于sessionAttributes动态权限
- **意图感知**：Evaluation模块Preview版本

### Microsoft Azure Entra Agent ID
- **上下文感知**：条件访问（Conditional Access）策略
- **身份传播**：工作负载身份联合（Workload Identity Federation）
- **身份归因**：Sign-in Logs记录完整审计链

### 火山Agent Identity（字节跳动）
- **核心机制**：入站与出站认证分离
- **入站认证**：验证用户身份（支持飞书、Google Identity等）
- **出站认证**：管理Agent访问下游服务及凭证
- **关键组件**：
  - Agent Workload Identity
  - 出站Gateway
  - TokenVault
  - 策略引擎（Cedar语言）

## 经典攻击向量

### 混淆代理人攻击（Confused Deputy）
- **场景**：低权限攻击者诱骗高权限Agent代表其执行操作
- **原因**：Agent使用单一高权限账号而非用户身份

### 提示词注入（Prompt Injection）
- **场景**：通过外部内容注入恶意指令
- **防御**：意图感知授权层分析操作与原始意图是否相符
