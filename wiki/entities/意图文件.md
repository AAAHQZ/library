# 意图文件 (Intent File)

> 不是代码，不是配置，不是文档——它们是给 AI Agent 的工作上下文，是人类意图的结构化表达，是软件项目里正在长出来的新一层。

## 简介

意图文件（Intent File）是指在项目根目录中存放的、专门写给 AI Coding Agent 阅读的 Markdown 文件。它们的核心功能是**将人类意图持久化、结构化地传递给 Agent**，使 Agent 每次工作前都能读到一致的上下文，而不是依赖一次性的 prompt 对话。

- **类型**：概念
- **领域**：AI 辅助开发、Agent 上下文工程、Harness Engineering

## 已知的意图文件类型

| 文件 | 用途 | 状态 |
|------|------|------|
| `CLAUDE.md` | 代码行为规范、架构约定 | 已广泛使用（Claude Code 原生支持） |
| `DESIGN.md` | UI 风格规范、设计系统 | 新兴（VoltAgent 模式，2026 年 5 月爆火） |
| `VOICE.md` | 产品语气和文案风格 | 预测中 |
| `DOMAIN.md` | 数据模型和业务规则 | 预测中 |
| 测试标准文件 | 测试策略和质量标准 | 预测中 |

## 核心特征

1. **持久性**：不随对话消失，存储在项目根目录中
2. **结构化**：Agent 能高效读取，优于 Figma 等可视化设计工具
3. **人类维护**：由人类编写和更新，传递的是真正的意图，不是 Agent 的猜测
4. **独立于具体工具**：纯 Markdown，任何 Agent 都能读取

## 设计意图与工程意图的分工

DESIGN.md 让设计意图持久化，CLAUDE.md 让工程意图持久化。两个文件放在项目根目录里，构成了 Agent 工作的稳定地基——不是靠更聪明的模型，而是靠更好的上下文设计。

## 与 Vibe Coding 的关系

意图文件是 Vibe Coding 从「随缘出活」走向「可控生产」的关键一步。Vibe Coding 的根本问题是人类意图没有稳定存放地点（prompt 是一次性的）。意图文件通过持久化、结构化的方式解决了这个问题。
