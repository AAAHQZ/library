# Context Engineering

> 上下文工程（Context Engineering）是指对 AI Agent 与大模型交互过程中上下文的组织、管理与注入方式进行系统性设计的工程实践，是 AI-Agent 从原型到生产的关键挑战之一。

## 核心概念

- **上下文窗口** — 大模型的输入容量限制，通常以 token 数量衡量
- **上下文注入** — 将外部信息（文档、代码、对话历史）嵌入 prompt 的技术
- **上下文压缩** — 对冗长上下文进行摘要、裁剪或结构化重组
- **分层上下文** — Warp 提出的三层上下文拆解：Global Context / Session Context / Turn Context
- **RAG** — Retrieval-Augmented Generation，通过检索增强生成能力

## 上下文工程在 AI-Agent 工程中的地位

上下文工程是 AI-Agent 工程实践的三大核心挑战之一（另外两个是 [[Agent隔离]] 和 [[Agent-Memory框架]]）。它直接影响：

1. **模型对任务的理解深度** — 上下文决定模型能否准确把握任务背景、约束条件和目标
2. **Token 消耗与成本** — 上下文占比直接影响推理成本和响应延迟
3. **任务执行准确性** — 错误的上下文注入导致模型产生幻觉或执行偏差

## 主要技术与方法

### 1. 分层上下文架构

典型分层设计（如 Warp 的三层模型）：

| 层级 | 范围 | 典型大小 | 更新频率 |
|------|------|----------|----------|
| Global Context | 项目级全局信息（代码库结构、架构文档） | 几十 KB | 低（周/版本级） |
| Session Context | 当前会话信息（用户偏好、当前任务状态） | 几 KB | 中（会话级） |
| Turn Context | 当前轮次信息（用户输入、即时任务） | < 1 KB | 高（每轮更新） |

### 2. 上下文压缩策略

- **摘要压缩** — 对长文本进行摘要，保留关键信息
- **结构化提取** — 从非结构化文本中提取结构化数据（JSON、表格）
- **选择性遗忘** — 对话历史中衰减不重要的信息
- **语义索引** — 使用向量数据库建立语义检索能力

### 3. RAG 与检索增强

RAG 通过外部知识库检索来增强模型上下文，解决：
- 大模型知识过时问题
- 长尾专业知识的注入
- 动态变化的企业内部数据

典型 RAG 流程：
1. **分块（Chunking）** — 将文档切分为可检索的片段
2. **向量化（Embedding）** — 使用嵌入模型将文本转为向量
3. **检索（Retrieval）** — 基于语义相似度检索相关片段
4. **注入（Injection）** — 将检索结果注入 prompt

### 4. 上下文注入模式

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| Prepend | 注入在用户输入之前 | 全局信息、角色设定 |
| Append | 注入在用户输入之后 | 工具输出、实时数据 |
| Sandwich | 重要信息放在首尾 | 需要强化记忆的关键内容 |
| Interleaved | 与用户输入交错 | 多步骤任务、链式推理 |

## 工程实践要点

### 上下文边界管理

- **容量规划** — 始终为用户输入和模型输出预留足够空间
- **优先级排序** — 当上下文接近上限时，按重要性排序保留内容
- **渐进式注入** — 先注入高频/高权重信息，再按需补充低频信息

### 格式与结构化

- 使用清晰的格式标记（XML tags、Markdown headers）
- 结构化输出格式约定（JSON Schema、TypeScript interfaces）
- 对复杂信息进行预处理，减少模型解析负担

### 质量与一致性

- 上下文版本控制 — 追踪上下文变更历史
- 自动化测试 — 用已知案例验证上下文注入效果
- A/B 对比 — 评估不同上下文策略的实际效果

### 格式与结构化（Dijkstra 视角）

Dijkstra（EWD 667, 1978）关于界面宽度的理论为上下文格式化工提供了理论解释：

- **用 XML 标签包裹输入（`<context>...</context>`）**：在自然语言中重建语法边界，将宽界面收窄
- **结构化输出格式约定**：在输出端重建形式结构
- **这些实践本质是在设计「一种临时性的、私有的形式化语言」**，并非真正用母语与机器对话

参见 [[界面宽度]] 和 [[形式符号]] 获取更多理论背景。
