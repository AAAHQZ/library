# EventHouse

> 阿里云的事件数据平台，定位为 AI 时代面向 Agent 的"标准插座"，旨在降低企业 Agent 接入真实业务世界的门槛。

## 简介

EventHouse 是阿里云推出的下一代事件数据平台，由阿里云智能集团高级技术专家沈林在 2026 GenAICon 上正式介绍公测。其核心定位是作为 Agent 与企业业务数据之间的"高速公路"，打通多源数据接入、语义对齐、知识沉淀、变更治理和 Serverless 体验。

- **类型**：工具/平台
- **领域**：云计算、AI Agent 基础设施、事件数据平台
- **发布方**：阿里云
- **状态**：公测中
- **相关概念**：[[环境工程]]、[[上下文供给]]、[[统一Catalog]]、[[知识对账]]、[[Agent制品化发布]]

## 详细内容

### 核心能力

EventHouse 围绕五大维度构建 Agent 上下文供给能力：

1. **信息感知**：三类信息感知方式——主动监听（Polling/Monitoring）、事件订阅（Event Subscription/EDA）、挂载查询（Mount Query）
2. **统一 Catalog**：管理 Agent 信息资产的统一目录，记录数据的语义、Schema、新鲜度、来源、适用范围和关联关系
3. **知识 Wiki**：基于 Catalog 语义信息和 Agent 业务设定（SOUL、Prompt、Gold Sample 等）构建可读、可审查、可持续迭代的知识库
4. **制品化发布**：将 Agent 更新封装为可管理的制品，支持发布前回归测试、发布中蓝绿发布、发布后可回滚
5. **Serverless 体验**：按量付费、秒级弹性、零运维

### 定位

EventHouse 想做的是 AI 时代面向 Agent 的"标准插座"：
- **广度**：打通消息系统、数据库、对象存储、SaaS 服务等多源数据接入
- **深度**：统一对齐结构化、半结构化和非结构化数据语义，构建知识 Wiki
- **流程**：把数据集成、存储、查询、检索整合为一体化服务
- **形态**：Serverless，按量付费、秒级弹性、零运维
