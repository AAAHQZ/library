# Qoder

> 阿里巴巴集团推出的 AI 编程平台/工具，已在淘天（淘天营销中后台）、高德（高德大模型应用平台）等多个团队落地实践，支持从需求 PRD 到直接部署的全流程自动化。

## 简介

Qoder 是阿里巴巴集团的 AI 编程平台，定位为"研发效能工具"，覆盖从需求分析、方案设计、代码生成到测试部署的全链路。与 [[Cursor]]、[[Claude Code]]、Codex 等通用 AI 编程工具不同，Qoder 深度集成阿里内部技术栈（如 Aone CI/CD 平台），并通过 MCP 协议打通运维链路。

## 关键信息

- **类型**：AI 编程平台/工具
- **所属**：阿里巴巴集团
- **领域**：AI 辅助研发、研发效能
- **核心能力**：知识库管理、Quest Spec 模式、专家团模式（Experts Mode）、Memory、Skills 扩展、MCP 集成

## 核心功能

### 1. 知识库管理（三层结构）
- **项目层**：项目概述、目录结构、架构设计、技术选型
- **技术层**：编码规范、中间件文档、最佳实践、常见问题
- **资产层**：可复用代码片段、组件、模板、历史需求 PRD

### 2. Quest Spec 模式
- 将需求 PRD 转化为结构化的 design.md 文档
- 包含数据模型、接口规范、验收标准
- 强调 HITL（Human-In-The-Loop，人在回路），AI 主动提问引导开发者澄清隐性知识

### 3. 专家团模式（Experts Mode）
- 大型任务拆解为可管理的小任务
- 按任务类型分配不同角色的 Agent（五类内置专家）
- Experts Leader 协调任务，人类可随时介入

### 4. Memory
- 结构化的方式来存储和管理长周期项目中的信息
- 解决 AI 的"上下文焦虑"问题

### 5. Skills 扩展机制
- AI 通过 Skills 获得额外能力（数据库操作等）
- 支持端到端的开发、测试、验证

### 6. MCP 集成
- 通过 Aone（阿里内部 CI/CD 平台）的 MCP 工具触发 CI/CD 流水线
- 打通从需求到部署的全链路

## 实践成果

- 高德团队：出码率 53% → 80-90%（2025.09 → 2026.05）
- 淘天营销中后台：AI 生码工作流最佳实践

## 不同素材中的观点

- **高德王树新（2026.05）**：Qoder 结合 SDD 和 Harness，打通了从需求到部署的全链路。开发者角色从编码者变为"需求澄清者、规范审核者、结果验证者"。
- **淘天营销前台技术团队（2026.04）**：通过 Qoder 的生码工作流统一收敛 AI 生码路径，提升可落地场景范围和实际效果。
