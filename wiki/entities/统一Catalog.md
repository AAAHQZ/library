# 统一Catalog（Unified Catalog）

> EventHouse 用于管理 Agent 信息资产的统一目录，提前记录并维护数据的语义、Schema、新鲜度、来源、适用范围和关联关系，解决 Agent 上下文"能不能被正确消费"的问题。

## 简介

统一 Catalog 是 EventHouse 的核心组件之一，类比为一个"图书馆馆藏目录"——Agent 需要的不只是更多信息，而是一份可以快速定位、持续更新、统一理解的信息资产目录。没有统一 Catalog，Agent 每次查询都像"考试前临时抱佛脚"：速度慢、Token 消耗高、容易产生语义偏差。

## 详细内容

### 统一 Catalog 管理的内容

- 数据的语义描述
- Schema 结构
- 数据新鲜度
- 来源信息
- 适用范围
- 关联关系

### 解决的问题

如果说信息感知解决的是"有没有上下文"，那么统一 Catalog 解决的就是"上下文能不能被正确消费"。它让 Agent 更清楚地知道：自己手里有哪些信息、每类信息意味着什么、需要时应该去哪里找、哪些内容值得优先使用。
