# Task-Driven（任务驱动）

> 由人编排和分派任务，AI 负责执行、拆解和留痕的 Agent 工作模式。

## 定义

Task-Driven 是 Agent 系统的基础工作模式：人先把值得做的事想好，送进系统，系统负责拆解、执行和留痕。人的角色是"项目经理 + 执行监督"，Agent 的角色是"执行器"。

## 特征

| 维度 | 说明 |
|------|------|
| 人的角色 | 项目经理 + 执行监督 |
| Agent 的角色 | 执行器 |
| 决策中心 | 在人脑子里 |
| 主要成本 | 人持续编排 |
| 适用场景 | 简单、一次性任务 |

## 局限

系统虽然能 24 小时执行，但高层的判断仍然依赖人：现在最值得做什么？哪个方向应该继续推进？遇到阻塞该换路径还是等待？Task-Driven 解决执行问题，但无法解决迭代问题。

## 从 Task-Driven 到 Goal-Driven

Task-Driven 是 Goal-Driven 的前提。一个连任务都执行不稳定、没有留痕、没有 Skill 沉淀的系统，不可能真的进入目标驱动。
