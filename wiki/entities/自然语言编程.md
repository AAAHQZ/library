# 自然语言编程

> 一种编程理念，主张用人类自然语言（英语、中文等）替代形式化编程语言来指挥计算机。Dijkstra 在 1978 年批评此理念，认为自然语言是「宽界面」，两侧都要付出额外代价。

## 历史

- **1978**：Dijkstra 撰写 EWD 667，批评自然语言编程理念
- **2022-2026**：LLM 与 AI Coding Agent 使「用人话写代码」成为日常工程实践
- **2026**：所有认真做事的团队都在想方设法把自然语言「变回」形式化的东西

## Dijkstra 的批判

Dijkstra 认为自然语言编程的问题在于：

1. **界面太宽**：自然语言包含大量歧义、隐含假设、上下文依赖，两端（人和机器）都要为处理这些歧义付出额外成本
2. **错觉自然**：母语的「自然」让人容易说出表面通顺实则无意义的话
3. **历史教训**：数学和科学的进步依赖于用形式符号替换自然语言

## 现代实践的反讽

2026 年的 LLM 工程实践显示了一个颇具讽刺意味的现象：

| 实践 | 实际效果 |
|------|---------|
| XML 标签包裹输入 | 在自然语言中重建语法边界 |
| 要求输出 JSON 而非自由文本 | 在输出端重建形式结构 |
| Spec-driven development | 在指令端重建形式化契约 |
| Few-shot examples | 压缩指令的歧义空间 |
| Tool use / Function calling | 将关键动作从描述退回为调用 |
| Constitutional AI | 在价值层补上形式化约束 |

本质上，我们并非真的在「用自然语言编程」，而是在用一种「伪装成自然语言的、经过人工收窄的准形式语言」编程。
