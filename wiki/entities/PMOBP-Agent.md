# PMOBP Agent

> 4.0阶段产物，一个Agent像专属PMO一样，按角色支持项目里所有成员，是信息中枢+任务执行者的结合体。

## 简介

PMOBP Agent是小红书PMO团队在4.0阶段（2026年Q1至今）打造的"项目专属BP"（Business Partner）。其核心定位是：**首先成为信息中枢，其次才是任务执行者**。架构上从"1人×1Session"升级为"1项目×N人×M Session"，为每个项目组提供一套共享的多渠道动态项目上下文。

- **领域**：项目管理 AI 化 / Agentic Engineering

## 详细内容

### 核心设计理念

不同角色问同一个问题，答案完全不同：
- 管理者问"项目进展怎样" → 想了解目标达成度、有没有卡点
- 项目负责人问 → 想了解全局核心进展和风险
- 项目成员问 → 可能只想知道自己依赖模块的进展

因此 PMOBP Agent 需要**角色感知能力**，自动调整回答粒度。

### 技术架构（4.0整体架构）

基于 [[OpenClaw]] 构建，复用其 Gateway、Session 模型、Multi-Agent 编排、IM 插件、Skill 体系、稳定性机制等基础能力，并接入 Knowledge 工程提供的三层渐进检索、定时知识整合、项目档案多源摄取等横向能力。

在此之上新增两层 PMOBP 专属能力：
1. **项目上下文路由**：识别用户在问哪个项目
2. **角色感知**：按管理者、项目负责人、项目成员自动调整回答粒度

### 已落地能力矩阵

| 类别 | 能力 |
|------|------|
| 信息查询 | 项目快照产出进展、按角色总结关注事项、查看KO/结项文档 |
| 资料生成 | 一键创建标准项目目录、项目管理各种模板获取、例会文档自动接续 |
| 协同操作 | PMOBP私发消息、群发消息（通知及催更）、机器人帮拉群并自动弹卡片绑定项目 |
| 项目管理 | Todo/进展增删查改、我的项目视图及项目常见问题解答、项目群消息&项目例会智能总结并写入知识库形成项目context |
