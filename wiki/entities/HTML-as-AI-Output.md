# HTML作为AI输出格式

> 相比 Markdown，HTML 能提供可视化锚点、实时交互、图形界面，但 Token 消耗更高、渲染时间更长。

## HTML vs Markdown 的基本判断

- **Markdown 是新时代的 Word**：简单、噪音少、人类看得懂、模型生成和修改顺手
- **HTML 是新时代的 PPT**：有样式、有布局、有图形、视觉舒服但 Token 消耗大

只有当 Token 消耗和生成时间不再是关键问题时，HTML 的价值才更大。

## 五个核心 Use Case

### 01 | 渲染方案对比画布
让 AI 生成 6 种截然不同的 UI 方案，以 HTML 网格形式并排展示。
- **提示词核心**：生成 N 种不同方法，在布局、语调和密度上都有差异，作为单个 HTML 文件以网格形式布局
- **本质**：把 HTML 当成可视化思考画布，所有可能性展示在面前来选择

### 02 | PR 解释器
HTML 里可以渲染 diff、加行内注释、画流程图、标颜色。
- **提示词核心**：把 diff 渲染出来，加上行内边注，按严重程度给发现的问题标颜色
- **扩展**：接手老项目让 AI 做 HTML 架构解释器；向非技术同事解释技术方案

### 03 | HTML 选择器（最推荐）
做带滑块的 HTML 调试器，让人在浏览器里实时调参，找到"刚刚好"的点后一键复制。
- **提示词核心**：给多个滑块和选项尝试不同效果，再给复制按钮把好参数复制出来
- **本质**：把 HTML 当成 AI 和人之间的协作界面
- **Claude Design 就是这个思路**

### 04 | 内容展示报告
HTML 报告有视觉锚点、图表、结构，能引导注意力。
- **提示词核心**：只读一遍就能看懂
- **区别**：HTML 报告是为读者优化的，Markdown 只是把信息倒出来

### 05 | 一次性操作工具
用 HTML 做专门为手头这一件事服务的临时编辑器，关键是有导出按钮。
- **场景**：拖拽排序工单优先级、功能开关配置编辑器、system prompt 实时预览
- **核心**：把文字难以表达的操作（颜色、缓动曲线、cron 表达式、正则、拖拽排序）变成图形界面
- **即用即抛，不用考虑复用和维护**

## 核心洞见

> HTML 在 AI 时代真正有意思的玩法，不是用来写文档，而是用来做一些以前你想都没想过的事情。

> 把 HTML 当成了 AI 和人之间的协作界面。AI 出代码，人调参数，最后参数再回到 AI 那里变成最终代码。
