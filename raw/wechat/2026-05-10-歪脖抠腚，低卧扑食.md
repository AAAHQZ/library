# 歪脖抠腚，低卧扑食

歪脖抠腚，低卧扑食
前半句"歪脖抠腚"是 Vibe Coding的 音译，歪着头挠着屁股，AI 就把活干了。爽是真的爽——但是Vibe Coding 时代的生产力狂欢，也意味着混乱的狂欢。
后半句"低卧扑食"就是 DevOps。趴低了干活，往实里扑，守住工程纪律的底线。
把这两半连起来，就是我想说的核心观点：DevOps 是 Vibe Coding 时代最后的护栏。
AI 把开发速度干到了 10 倍，但如果交付还是慢得不行，如果线上事故频发、代码无人维护，那 10 倍的速度只会制造 100倍的混乱。DevOps 的工程纪律——可重复的构建、可观测的部署、可追溯的变更——在 AI 时代不是过时了，而是比任何时候都重要。
我是谁
六年 DevOps经验，一直在这条线上：CI/CD、平台工程、基础设施自动化、研发效能。从零搭过流水线，建过内部平台，踩过的坑大概可以铺一条路。
近一年开始把注意力转向 AI —— 准确说，是不得不、半强制半自愿、被时代浪潮所裹挟、被迫地转向AI。追踪的方式很原始：看到值得看的文章、报告、项目，存下来；遇到自己解决了的问题，记下来；想清楚一件事了，写出来。这个号就是这些笔记的出口。
这个号是干嘛的
这个号想做的就两件事：
分享个人的思考和经验 ——做了六年 DevOps，从零搭过流水线、建过平台、踩过坑、也填过坑。这些经历里长出来的判断，比纯理论更有参考价值。
介绍业界现在的最佳实践 ——自己的经验有边界，所以也在持续跟踪行业里真正跑通的做法。看到好的就消化进来，结合自己的理解再输出。不是什么"硅谷最新范式"的转发机，是把有价值的东西掰开揉碎了再端上来。
两条线交汇的地方，就是AI 时代的工程实践怎么落地。
为什么写
学习和实践的过程中有几个问题反复出现，我觉得值得拿出来聊。
Vibe Coding 到底靠不靠谱？
Vibe Coding 很爽，这个不否认。歪着脖子哼几句需求，代码就出来了。但代价也很实在：代码量爆炸、没人理解全局、技术债涨得比写代码还快。
整个行业正在从"哇，AI 能写代码"的兴奋期，滑进"AI 写的代码谁来维护"的阵痛期。从 Vibe Coding 到 Harness Engineering 的转向，说白了就是把 Agent 当"牲口"管，不是当"宠物"养——可替代、可审计、有边界、可观测。
DevOps 在 AI 时代还有没有用？
有，而且更有用。
Agent 开始写代码、部署代码、修复代码之后，CI/CD 变成了 Agentic CI/CD，从脚本自动化升级成智能自治。但底层那些工程纪律，构建可重复、部署可观测、变更可追溯，不但没消失，反而成了最后的护栏。
自动化测试、灰度切流、秒级回滚，是 AI 时代最后的尿不湿，最后的护工。你也不想歪脖抠腚抠出一坨那啥糊一手吧。
"低卧扑食"
低卧扑食是一个姿态——不装。
不会给你讲"AI 将重构一切"、“AI淘汰程序员”这种屁话。我只能讲清楚一些我个人眼中的看法（不一定对），什么工程实践对我有用、什么时候有用、有哪些坑；分享一些看到的好东西；偶尔扔点灵感和思考出来。
以及，
我作为一个小兵，卧着在地上看着公司内外环境和领导的一些阴暗看法。
你会在这里看到什么
Harness Engineering：如何驾驭大模型
Agentic CI/CD：Agent 驱动的流水线长什么样，怎么从"自动执行"进化到"智能自治"，中间差了什么
平台工程：内部开发平台的构建，监控、稳定性如何建设
AI 编程实践：Vibe Coding 爽在哪、痛在哪，工程纪律怎么建
工程管理：AI 把开发速度干到十倍之后，为什么交付还是慢；人才密度、认知债、判断力依赖这些问题怎么破
工具和框架：MCP、Qoder、Spec 驱动开发、Context Engineering——这些到底在解决什么问题
每篇背后要么有具体的工程实践，要么来自真实案例。是我猜的，我会说"这是我猜的"。
"低卧扑屎"最后
名字还有一层意思没说。
"扑食"念快了像"扑屎"。也行——AI 时代信息这么多，可能我正在扑向一坨屎，扑就扑了。低卧扑食这名字主要是提醒自己：别飘，趴下来，往实里扑。
如果你也在做工程、写代码、搞 DevOps，或者在跟 AI Agent 打交道，欢迎一起看看这个行业里到底在发生什么。
不是每个新东西都值得追，但那些真正改变做事方式的变化，值得搞清楚。
